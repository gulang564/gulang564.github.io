## 读谷歌的三篇重要论文的观后感读谷歌的三篇重要论文的观后感

    一个技术的由来，都是有它的起因的，谷歌的分布式管理技术也不例外，这项技术的产生，是由3个美国的三个大学生的奇思妙想（其实就是没有经费买好的服务器，被逼无奈才想出来的奇妙的想法）经费不足，买不起好的服务器，只有买二手的，好，买二手的机器就容易坏，坏了的话数据就会丢失，做到事情就白费了，所有分布式管理的想法应运而生。分多处进行数据的存储，即使一个位置的数据出问题，还有备份，（不会运气那么差，3处数据都丢失吧）。这个想法提出后过了几年才被谷歌应用并得到发展。

Google对大数据问题的思想
1、GFS：Google File System
       对数据进行分布式文件系统的存储，解决的大数据的存储问题
    好处：
        与创建失败的无效Chunk一致的处理方式；
        批量执行开销分散，Master相对空闲时进行；
        删除操作一定时间内可逆转。
    坏处：
        不便于用户进行 存储空间调优。
解决方案：
        再次删除加速回收，不同命名空间不同复制回收策略。

2、MapReduce
      最初是因为Google要去爬取全球的网页然后对其进行排名(PageRank)而衍生出的问题，一种用于大数据计算的模型，核心思想是大事化小处理，最后在合并结果，先拆分再合并。MapReduce 编程模型的原理是：用户自定义的 Map 函数接受一个输入的 key/value pair 值，然后产生一个中间 key/value pair 值的集合。MapReduce 库把所有具有相同中间 key 值 I 的中间 value 值集合在一起后传递给 reduce 函数。用一个例子详细解释mapreduce思想。要求数出1000张纸牌中有多少张黑桃，100个人每人随机发放10张，每个人数出自己拿到的牌中有多少张黑桃，这是map函数实现的，reduce函数的作用是将100人数的张数加在一起输出。Key是黑桃，value是黑桃的张数。

3、BigTable
      采用NoSQL数据库将数据存在一张大表之中，通过牺牲存储的空间来换取性能 每个Table都是一个多维的稀疏图，Table有行和列组成，并且每个存储单元cell都有一个时间戳，在不同的时间对同一个存储单元cell有多分拷贝，这样就可以记录数据的变动情况

主要的应用技术就是主要的应用技术就是

       利用HDFS技术，在服务器端分布式存储，，客户端上传的文件分为3份，放在3个不同的位置，有一个机器是管理员，（NameNode），还有一个管理员的备胎（管理员挂掉了，就有它来管理剩下的机器处理数据）名字叫SecondaryNameNode，其他的机器接受分来的不同的数据进行处理，但两个管理员都挂掉了，就有剩余的这些机器进行选举，在选一个机器进行管理，Master会通过周期性的心跳信息监控Chunck服务器的状态，看服务器是否还活着。如果服务器挂掉，则把服务器的所有数据回收，交个其他服务器处理（所有数据都回收包括已近挂掉服务器已近处理了的数据）
